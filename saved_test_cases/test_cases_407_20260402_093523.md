# AI Generated Test Cases

## Feature Context
> User Story: As a registered user, I want to reset my password if I forget it so that I can regain access to my account.  Acceptance Criteria:  Request Reset  User clicks “Forgot Password” on login page  User submits registered email address  System displays generic success message (to prevent email enumeration)  Email Handling  System sends password reset email with secure link  Email contains:  Reset link  Expiry information (e.g., valid for 15 minutes)  Email is sent only if account exists (silently ignore otherwise)  Token & Security  Reset link contains a unique, encrypted token  Token:  Is single-use  Expires after configured time  Is invalidated after successful reset  Reset Password  User clicks link → navigates to reset page  User enters:  New password  Confirm password  Password must meet policy:  Minimum 8 characters  At least 1 uppercase, 1 number, 1 special character  Post Reset  Password is updated successfully  User is redirected to login page  All existing sessions are invalidated  Confirmation message is shown  Error Handling  Expired/invalid token → show proper error message  Password mismatch → validation error  Weak password → show policy guidance   Security Requirements  Use hashed tokens (not plain text)  Rate limit reset requests (e.g., 5 per hour per user/IP)  Prevent user enumeration (same response for valid/invalid emails)  Enforce HTTPS for all reset links  Log all reset attempts for auditing  Test Scenarios  Positive Cases  Valid email → reset link sent  Valid token → password successfully reset  Negative Cases  Invalid email → no system disclosure  Expired token → reset denied  Reused token → blocked  Weak password → rejected
