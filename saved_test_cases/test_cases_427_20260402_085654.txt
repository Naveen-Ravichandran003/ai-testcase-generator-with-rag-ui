ID: TC-001
Title: Valid Email Submission for Password Reset
Description: Verify that a valid email submission triggers a password reset email.
Preconditions: User is registered with a valid email address.
Steps:
1. User clicks the "Forgot Password" button on the login page.
2. User submits a valid registered email address.
3. User waits for the password reset email.
Expected Result: System displays a generic success message and sends a password reset email with a secure link.
Priority: High
Test Type: positive/happy path scenarios and basic functionality

Preconditions: Two registered users, User A and User B, both have valid tokens for password reset. User A's session is invalidated due to inactivity.
Steps:
1. User A clicks the reset link and navigates to the reset page.
2. User A enters a weak password and submits the form.
3. User B attempts to reset their password while User A's token is still valid.
4. User A's session is invalidated due to inactivity.
Expected Result: User B is shown an error message indicating that the token has expired.
Priority: High
Test Type: concurrent user and race condition scenarios
